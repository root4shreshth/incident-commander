IncidentCommanderEnv

An OpenEnv RL environment where AI agents act as on-call Site Reliability Engineers, diagnosing and remediating production incidents across a simulated microservices cluster.

OpenEnv Compatible SRE / DevOps Domain Baseline: 0.93 avg

How It Works

An AI agent goes through the standard RL loop: observe, reason, act, learn

🚨
1. Alert Fires
Agent receives PagerDuty-style incident alert with affected services
→
🔍
2. Investigate
Read logs, check metrics, run diagnostics across 9 microservices
→
🔬
3. Diagnose
Trace dependency chains to identify root cause
→
⚡
4. Remediate
Restart, rollback, scale, or reconfigure to fix the issue

Simulated Infrastructure

9 interconnected microservices with realistic failure propagation

🌐 Service Topology

frontend-bff --> api-gateway |--> order-service --> payment-service | --> inventory-service | --> postgres-db |--> user-service ---> auth-service |--> notification-service

📊 What Each Service Has

  • ● Health states: healthy, degraded, unhealthy, crashed
  • ● Live metrics: CPU%, memory, latency p50/p99, error rate
  • ● Structured logs: timestamped with realistic patterns
  • ● Deployment history: versions with rollback capability
  • ● Dependencies: upstream/downstream failure cascading
  • ● Config: memory limits, replicas, DB pool size

API Reference

Base URL: https://hype4raj-incident-commander-env.hf.space

POST /reset Start a new incident episode 
Request:
{ "task_id": "oom_crash" | "db_pool_exhaustion" | "bad_deployment_cascade" }

Response:
{
  "observation": { "message": "INCIDENT ALERT...", "alert": "...", "dependency_graph": {...} },
  "reward": 0.0,
  "done": false,
  "info": { "task_id": "oom_crash", "max_steps": 15 }
}
POST /step Execute an SRE action 
Request:
{
  "action_type": "read_logs",
  "target_service": "payment-service",
  "parameters": { "lines": 50, "severity": "ERROR" }
}

Response:
{
  "observation": { "message": "Logs for payment-service...", "logs": [...] },
  "reward": 0.03,
  "done": false,
  "info": { "step_count": 2 }
}
GET /state Get current episode state 
Response:
{
  "episode_id": "uuid",
  "step_count": 5,
  "task_id": "oom_crash",
  "incident_resolved": true,
  "current_score": 1.0,
  "max_steps": 15
}
GET /tasks List available incident scenarios 
Response:
{
  "tasks": {
    "oom_crash": { "difficulty": "easy", "max_steps": 15 },
    "db_pool_exhaustion": { "difficulty": "medium", "max_steps": 25 },
    "bad_deployment_cascade": { "difficulty": "hard", "max_steps": 35 }
  }
}
GET /health Liveness check 
Response: { "status": "ok" }
